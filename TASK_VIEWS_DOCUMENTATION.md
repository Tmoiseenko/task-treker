# Документация по Blade представлениям задач

## Обзор

Созданы все необходимые Blade представления для управления задачами в соответствии с требованиями 3.1, 9.1-9.6.

## Созданные файлы

### 1. Layout (resources/views/layouts/app.blade.php)
Базовый layout приложения с:
- Навигацией
- Flash-сообщениями (success/error)
- Подключением Tailwind CSS через Vite
- Адаптивным дизайном

### 2. Список задач (resources/views/tasks/index.blade.php)
**Функциональность:**
- Отображение списка задач с пагинацией
- Фильтры по:
  - Поиску (название и описание)
  - Проекту
  - Статусу
  - Приоритету
  - Исполнителю
  - Автору
- Визуальные индикаторы:
  - Бейджи приоритета (высокий/средний/низкий/заморожено)
  - Бейджи статуса (не выполнено/в работе/на тестировании/тест провален/выполнено)
  - Красный фон для просроченных задач
  - Теги с цветами
- Кнопка "Создать задачу" (с проверкой прав)

**Требования:** 3.1, 9.1, 9.2, 9.3, 9.4, 9.5, 9.6

### 3. Создание задачи (resources/views/tasks/create.blade.php)
**Поля формы:**
- Название* (обязательное)
- Описание
- Проект* (обязательное, выпадающий список)
- Приоритет* (обязательное, выпадающий список)
- Срок выполнения (date picker)
- Исполнитель (опциональный, выпадающий список)
- Теги (множественный выбор)

**Функциональность:**
- Валидация на стороне клиента (required поля)
- Отображение ошибок валидации
- Сохранение старых значений при ошибке (old())
- Кнопки "Создать задачу" и "Отмена"

**Требования:** 3.1, 9.1

### 4. Редактирование задачи (resources/views/tasks/edit.blade.php)
**Функциональность:**
- Те же поля, что и в форме создания
- Предзаполнение текущими значениями задачи
- Обновление через PUT запрос
- Кнопки "Сохранить изменения" и "Отмена"

**Требования:** 3.1, 9.1

### 5. Детальная страница задачи (resources/views/tasks/show.blade.php)
**Основной контент:**
- Заголовок с названием задачи
- Бейджи приоритета и статуса
- Описание задачи
- Этапы задачи (TaskStages)
- Комментарии с автором и временем
- Прикрепленные файлы
- Связанные баг-репорты
- История изменений (audit log)

**Боковая панель:**
- Действия:
  - "Взять в работу" (для неназначенных задач со статусом TODO)
  - "Отправить на тестирование" (для задач в работе)
  - "Завершить задачу" (для задач на тестировании)
- Информация о задаче:
  - Проект
  - Автор
  - Исполнитель
  - Срок выполнения (с индикатором просрочки)
  - Даты создания и обновления
- Теги
- Документы
- Чек-лист с прогресс-баром

**Кнопки управления:**
- "Редактировать" (с проверкой прав)
- "Удалить" (с подтверждением и проверкой прав)

**Требования:** 3.1, 9.1, 9.2, 9.3, 9.4, 9.5, 9.6

## Визуальные особенности

### Цветовая схема приоритетов:
- **Высокий**: красный (bg-red-100 text-red-800)
- **Средний**: желтый (bg-yellow-100 text-yellow-800)
- **Низкий**: зеленый (bg-green-100 text-green-800)
- **Заморожено**: серый (bg-gray-100 text-gray-800)

### Цветовая схема статусов:
- **Не выполнено**: серый (bg-gray-100 text-gray-800)
- **В работе**: синий (bg-blue-100 text-blue-800)
- **На тестировании**: фиолетовый (bg-purple-100 text-purple-800)
- **Тест провален**: красный (bg-red-100 text-red-800)
- **Выполнено**: зеленый (bg-green-100 text-green-800)

### Индикаторы просрочки:
- Просроченные задачи выделяются красным фоном (bg-red-50)
- Дата просрочки отображается красным цветом с пометкой "Просрочено"

## Технические детали

### Используемые технологии:
- **Tailwind CSS** - для стилизации
- **Blade Templates** - шаблонизатор Laravel
- **Alpine.js** - готов к использованию для интерактивности (подключен через Vite)

### Маршруты:
- `GET /tasks` - список задач (tasks.index)
- `GET /tasks/create` - форма создания (tasks.create)
- `POST /tasks` - создание задачи (tasks.store)
- `GET /tasks/{task}` - детальная страница (tasks.show)
- `GET /tasks/{task}/edit` - форма редактирования (tasks.edit)
- `PUT /tasks/{task}` - обновление задачи (tasks.update)
- `DELETE /tasks/{task}` - удаление задачи (tasks.destroy)
- `POST /tasks/{task}/take` - взять задачу в работу (tasks.take)
- `POST /tasks/{task}/assign` - назначить задачу (tasks.assign)
- `POST /tasks/{task}/change-status` - изменить статус (tasks.change-status)

### Авторизация:
Все представления используют Laravel Policies через директивы `@can`:
- `@can('create', App\Models\Task::class)` - для кнопки создания
- `@can('update', $task)` - для кнопки редактирования
- `@can('delete', $task)` - для кнопки удаления
- `@can('take', $task)` - для кнопки "Взять в работу"
- `@can('changeStatus', [$task, $newStatus])` - для изменения статуса
